## 1. 系統概述

### 1.1 專案背景

隨著太陽能發電的普及，工業大型太陽能儲能系統的規模不斷擴大。為有效管理分散式電站的運維工作，需要一套整合性的管理平台，涵蓋客服工單、設備監控告警、維修派工等核心功能。

### 1.2 系統目標

- **集中化管理**：整合多個工業電站的設備資訊與運維流程
- **快速反應**：即時監控設備狀態，自動產生告警工單
- **流程標準化**：建立標準化的客服與維修作業流程
- **數據可視化**：提供完整的報表與儀表板，輔助決策

### 1.3 系統範圍

**包含：**
- 客服工單管理系統
- 設備監控與告警系統
- 維修派工管理系統
- 備品庫存管理（基礎版）
- 數據分析與報表系統

**暫不包含：**
- 外部 SCADA/EMS 系統整合（後續階段）
- 移動端 App（Web 響應式設計優先）

## 2. 使用者角色定義

### 2.1 角色總覽

| 角色 | 簡寫 | 說明 |
|------|------|------|
| 客服專員 | CS | 受理客戶回報、建立工單、追蹤進度 |
| 運維工程師 | OE | 監控設備狀態、處理告警、技術支援 |
| 維修技術員 | MT | 現場維修作業、回報維修結果 |
| 客戶/業主 | CL | 查看設備狀態、提交服務請求 |
| 系統管理員 | SA | 使用者管理、系統設定、權限控管 |

### 2.2 角色詳細說明

#### 🎧 客服專員 (Customer Service)

**主要職責：**
- 受理客戶回報的設備異常或服務需求
- 建立、分派、追蹤客服工單
- 與客戶保持溝通，更新工單狀態
- 生成客服相關報表

**系統權限：**
- ✅ 建立/查看/編輯工單
- ✅ 分派工單給運維工程師或技術員
- ✅ 查看設備基本資訊
- ❌ 無法修改系統設定

#### 🔧 運維工程師 (Operations Engineer)

**主要職責：**
- 監控所有電站的設備運行狀態
- 處理系統自動產生的告警工單
- 執行預防性維護計畫
- 提供技術支援與問題診斷

**系統權限：**
- ✅ 查看所有設備即時數據
- ✅ 處理告警與建立維護工單
- ✅ 查看完整設備歷史紀錄
- ✅ 制定預防性維護計畫

#### 🛠️ 維修技術員 (Maintenance Technician)

**主要職責：**
- 接收派工的現場維修任務
- 執行設備檢修與故障排除
- 回報維修結果與使用備品
- 拍攝現場照片作為紀錄

**系統權限：**
- ✅ 查看指派給自己的工單
- ✅ 更新維修進度與結果
- ✅ 上傳現場照片與文件
- ❌ 無法查看其他技術員的工單

#### 🏢 客戶/業主 (Customer/Owner)

**主要職責：**
- 查看所屬電站的設備運行狀態
- 提交服務請求或回報異常
- 查看歷史維修紀錄與報表

**系統權限：**
- ✅ 查看自己名下的電站資訊
- ✅ 提交服務請求
- ✅ 查看歷史工單與維修紀錄
- ❌ 無法修改其他客戶的資料

#### ⚙️ 系統管理員 (System Administrator)

**主要職責：**
- 管理使用者帳號與權限
- 系統設定與參數調整
- 備份與還原管理
- 系統監控與效能優化

**系統權限：**
- ✅ 完整系統管理權限
- ✅ 使用者與角色管理
- ✅ 系統設定與參數調整
- ✅ 資料備份與還原

## 3. 功能需求

### 3.1 模組總覽

```
太陽能儲能管理系統
├── A. 使用者認證與權限管理
├── B. 儀表板與數據可視化
├── C. 客服工單管理系統
├── D. 設備監控與告警系統
├── E. 維修派工管理系統
├── F. 備品庫存管理（基礎版）
└── G. 數據分析與報表系統
```

### 3.2 模組詳細需求

#### A. 使用者認證與權限管理

**A1. 使用者註冊與登入**
- 支援帳號密碼登入
- 支援 LDAP/AD 企業網域整合（後續）
- 雙因素認證（2FA）支援

**A2. 角色與權限管理**
- 五種預設角色（CS, OE, MT, CL, SA）
- 自訂角色建立功能
- 細粒度權限控制（按模組、按操作）

**A3. 密碼管理**
- 密碼複雜度要求
- 定期更換提醒（90天）
- 密碼遺失找回機制

#### B. 儀表板與數據可視化

**B1. 總覽儀表板（管理員/運維工程師）**
- 即時設備運行狀態總覽
- 今日工單統計（新增/處理中/已完成）
- 告警列表與嚴重程度分級
- 電站發電量即時曲線

**B2. 客戶儀表板（客戶/業主）**
- 所屬電站狀態總覽
- 歷史發電量統計圖表
- 進行中工單列表
- 設備健康度評分

**B3. 技術員儀表板（維修技術員）**
- 今日指派工單列表
- 緊急工單提醒
- 導航至維修地點（地圖整合，後續）

#### C. 客服工單管理系統 ⭐核心功能

**C1. 工單建立**
- 手動建立工單（客服專員）
- 自動產生工單（系統告警觸發）
- 客戶線上提交服務請求

**C2. 工單欄位定義**
```yaml
工單編號: 自動生成 (WO-YYYYMMDD-NNNN)
標題: 文字輸入（必填）
類別: 下拉選單 [設備故障/定期保養/諮詢/其他]
優先級: 下拉選單 [緊急/高/中/低]
描述: 多行文字（必填）
設備ID: 關聯設備（選填）
電站: 關聯電站（選填）
聯絡人: 自動帶入或手動輸入
聯絡電話: 文字輸入
期望到訪時間: 日期時間選擇器
附件: 圖片/文件上傳（選填）
狀態: 自動/手動更新 [待處理/處理中/已完成/已關閉]
```

**C3. 工單狀態流程**
```
[新建] → [已分派] → [處理中] → [待確認] → [已完成] → [已關閉]
   ↑                                                        |
   └──────────────── 退回重做 ←──────────────────────────────┘
```

**C4. 工單分派**
- 手動分派給指定運維工程師或技術員
- 自動分派規則（按電站區域、技術專長）
- 分派通知（Email/系統通知）

**C5. 工單追蹤**
- 完整操作紀錄（誰、何時、做了什麼）
- 時效監控（SLA 達成率）
- 逾期工單自動提醒

#### D. 設備監控與告警系統 ⭐核心功能

**D1. 設備管理**
- 設備基本資料維護（型號、規格、安裝日期、保固期限）
- 設備分級管理（電站 → 機組 → 設備）
- 設備狀態標記（正常/異常/維修中/停用）

**D2. 即時監控**
- 發電功率、電壓、電流等即時數據顯示
- 電池儲能狀態（SOC、SOH）
- 溫度、濕度等環境參數

**D3. 告警管理**
- 告警等級分級：
  - 🔴 **緊急**：設備停機、安全風險（5分鐘內回應）
  - 🟠 **高**：效能大幅下降（30分鐘內回應）
  - 🟡 **中**：參數異常但未停機（2小時內回應）
  - 🟢 **低**：預警資訊，需關注（24小時內回應）

- 告警觸發條件自訂
- 告警通知機制（Email、SMS、系統通知）
- 告警歷史紀錄查詢

**D4. 預警機制**
- 設備效能衰退趨勢分析
- 預防性維護提醒（按時間或運行小時數）
- 備品庫存不足預警

#### E. 維修派工管理系統 ⭐核心功能

**E1. 任務指派**
- 手動建立維修任務
- 由工單自動轉換為維修任務
- 排程管理（日期、時間、地點）

**E2. 技術員工作負載**
- 查看每位技術員的任務列表
- 避免過度派工（負載均衡）
- 技能匹配建議

**E3. 現場作業支援**
- 設備詳細資訊與歷史維修紀錄
- 標準作業程序（SOP）查閱
- 現場照片/影片上傳

**E4. 維修完成確認**
- 填寫維修報告（故障原因、處理方式、使用備品）
- 客戶簽核確認（電子簽名，後續）
- 工單自動關閉流程

#### F. 備品庫存管理（基礎版）

**F1. 庫存管理**
- 備品明細維護（型號、規格、存量、最低庫存）
- 出入庫紀錄追蹤
- 低庫存自動提醒

**F2. 領用管理**
- 維修任務關聯備品領用
- 自動扣減庫存
- 報表統計

#### G. 數據分析與報表系統

**G1. 預設報表**
- 每日/每週/每月發電量統計
- 設備故障率分析
- 工單處理時效統計（SLA 達成率）
- 技術員績效評分

**G2. 自訂報表**
- 選擇時間範圍、電站、設備類型
- 匯出格式：PDF、Excel、CSV

**G3. 儀表板自訂**
- 拖曳式元件排列
- 儲存個人化佈局

## 4. 非功能需求

### 4.1 效能需求

| 項目 | 要求 |
|------|------|
| 頁面載入時間 | ≤ 3秒（一般頁面）、≤ 5秒（報表頁） |
| API 回應時間 | ≤ 500ms（95%請求） |
| 同時線上使用者 | ≥ 100人 |
| 設備數據更新頻率 | ≤ 5秒/次 |
| 系統可用性 | ≥ 99.5%（每月非預期停機 ≤ 3.6小時） |

### 4.2 安全性需求

- **資料加密**：傳輸層 TLS 1.2+，敏感資料資料庫加密
- **存取控制**：基於角色的存取控制（RBAC）
- **稽核紀錄**：所有關鍵操作記錄不可篡改的稽核日誌
- **資安掃描**：定期進行漏洞掃描與滲透測試
- **資料備份**：每日自動備份，保留30天

### 4.3 可維護性需求

- 模組化架構，各功能模組獨立部署
- API 文件自動生成（Swagger/OpenAPI）
- 日誌集中管理，支援搜尋與告警
- 程式碼覆蓋率 ≥ 80%

### 4.4 可擴展性需求

- 支援水平擴展（增加伺服器處理更多電站）
- 資料庫分區策略，支援百萬級設備數據
- API 版本管理，確保向後相容

### 4.5 瀏覽器相容性

| 瀏覽器 | 版本要求 |
|--------|----------|
| Chrome | 最新版 - 2 |
| Firefox | 最新版 - 2 |
| Safari | 最新版 - 1 |
| Edge | 最新版 - 2 |

### 4.6 響應式設計

- 桌面版：1920×1080 及以上
- 平板版：768px - 1024px
- 手機版：320px - 767px

## 5. 系統架構建議

### 5.1 技術選型

| 層級 | 技術選擇 | 理由 |
|------|----------|------|
| **前端** | React + TypeScript | 生態系豐富、TypeScript 類型安全 |
| **UI 框架** | Ant Design / Material-UI | 元件庫完整、企業級應用適合 |
| **後端** | Node.js (NestJS) / Python (FastAPI) | 開發效率高、生態系完善 |
| **資料庫** | PostgreSQL + TimescaleDB | 關聯式資料 + 時間序列數據（設備監控） |
| **快取** | Redis | 高速讀取、Session 管理 |
| **訊息佇列** | RabbitMQ / Redis Streams | 非同步任務處理、告警通知 |
| **容器化** | Docker + Docker Compose | 本地部署、環境一致 |
| **反向代理** | Nginx | 靜態檔案服務、負載平衡 |
